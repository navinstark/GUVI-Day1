1.1 Workflow:

1.Request:

Client sends a request for each resource over a separate TCP connection.

2.Processing:

Server processes requests sequentially and sends responses back over the same connection.

3.Resource Retrieval:

Client parses HTML for additional resources and initiates separate requests for each.

HTTP/2 Workflow:

1.Request:

Client sends multiple requests over a single TCP connection.

2.Processing:

Server processes requests concurrently, utilizing multiplexing.

3.Resource Retrieval:

Client receives responses in parallel, benefiting from header compression and server push for optimal resource delivery.

Conclusion

HTTP/2 offers several enhancements over HTTP/1.1, including multiplexing, header compression, and server push, which collectively improve performance, reduce latency, and enhance the user experience. By optimizing resource delivery and communication between clients and servers, HTTP/2 aims to address the limitations of its predecessor and support the evolving needs of modern web applications.
